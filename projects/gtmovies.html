Overview

GT Movies Store is a full-stack web application developed using Django 5, built alongside the textbook Django 5 for the Impatient by Daniel Correa and Greg Lim. The application simulates an online movie marketplace where users can browse a curated library, manage purchases, and interact through reviews, while administrators maintain full control over platform data and moderation.

Deliberate approach I typed every line of code by hand rather than copying from the book's GitHub repository. By the middle stages I could anticipate upcoming steps without consulting the instructions — the patterns had clicked. The review reporting feature was not covered in the textbook at all; I designed and implemented it independently, which proved I had genuinely internalized the framework.

All 21 Features

  • View application information through an About page
  • Register a new account and get redirected to login on completion
  • Log in and see authenticated status reflected in the navigation bar
  • Browse the full movie catalog with images and titles
  • Search movies dynamically by title keyword
  • Access the shopping cart at any time through the persistent nav
  • Add movies to cart by selecting a quantity on the detail page
  • Purchase movies from cart — generates a unique order on checkout
  • Clear the cart entirely with a dedicated Remove All button
  • Create movie reviews as a logged-in user, with username and timestamp
  • Edit your own reviews with ownership control enforcement
  • Delete your own reviews immediately
  • View all reviews without needing to be logged in
  • Full movie detail pages with description, price, image, and reviews
  • Complete order history with numbers, dates, items, quantities, and totals
  • Hosted on PythonAnywhere — accessible from any desktop browser
  • Responsive layout that adapts to tablets and smartphones
  • Admin: manage users via the Django admin interface
  • Admin: add, update, and delete movies from the catalog
  • Admin: create, edit, or delete any review for full moderation
  • Report inappropriate reviews — removes them from public view immediately (custom feature, not in textbook)

What I Learned

This was my first substantial Django application. The choice to hand-type every line paid off — by the end I could reason about Django's URL routing, view logic, template inheritance, and ORM without needing to reference the book. The independently built reporting feature proved I had internalized the framework rather than just followed instructions.
