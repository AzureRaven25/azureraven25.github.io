Overview

SACC Hoops March Madness is a full-stack web application built for the St. Alphonsa Syro-Malabar Forane Catholic Church's annual basketball tournament in Atlanta, Georgia. The site serves as a central hub for players, team captains, and fans to follow the tournament in real time - from registration all the way through the final championship game. It is designed to feel like a professional sports platform while being tailored specifically to the SACC community.

The platform covers the entire tournament lifecycle: players can register and submit payment via Zelle, captains can manage their rosters, and everyone can follow live scores, standings, and player stats as games are played. A fantasy points system (FPTS) adds a competitive layer on top of the real tournament, letting fans track individual player performance across the bracket. This project was built entirely independently - no starter templates, no component libraries - as a real deployed product used by the actual SACC community.

The Problem It Solves

Community sports tournaments often rely on group chats, spreadsheets, and word-of-mouth to track standings and schedules. SACC's annual basketball tournament had the same problem - no central place for players to register, fans to follow scores, or captains to manage rosters. I built this platform to replace all of that with a single, live, mobile-friendly web app that anyone in the community can open on their phone during games.

Key Features

Live Score Ticker
A scrolling score ticker runs across the top of every page showing live and recent game scores, updating in real time via Supabase Realtime.
Playoff Bracket
A full bracket view with auto-advancing results and live game status - updates automatically as scores are entered by the admin.
Team Pages
Each team has a dedicated page with full rosters, per-player stats, team colors, and record. Captains can manage their roster through the platform.
Fantasy Points System (FPTS)
A custom formula (PTS + REB x1.2 + AST x1.5 + STL x3 + BLK x3 - TO) ranks all players across the tournament, giving fans a way to follow individual performance.
Player Stat Leaderboards
Sortable leaderboard tables for FPTS, points, rebounds, assists, and steals. Every column is clickable to re-sort live.
Player Profile Pages
Individual profile pages per player featuring radar charts and stat bars built with Recharts, showing their performance breakdown across all categories.
Power Rankings
Weekly power rankings with movement arrows (up, down, same) showing team momentum across the tournament.
Pick'em Predictions
Fans can make predictions on upcoming games and see live pick percentages across all users - no login required, tracked by anonymous browser session ID.
MVP Voting
Anonymous one-vote-per-session MVP voting system. Results update in real time as votes come in throughout the tournament.
Snake Draft Board
An interactive snake draft board with a pick timer and real-time roster updates - used by team captains to select players before the tournament.
Online Registration
Player registration form with integrated Zelle payment flow and waiver tracking. Submission data feeds directly into the admin panel.
Admin Panel
A full admin interface for managing games, players, teams, and commissioner announcements - everything needed to run the tournament from one dashboard.
Player & Team of the Week
Weekly spotlights on the home page highlighting standout individual and team performances from the previous round.
Game Schedule
Full schedule view showing court assignments, tip-off times, round info, and live game status for every game in the tournament.
GroupMe Integration
Commissioner announcements sync with GroupMe so updates reach the whole community instantly through the messaging app they already use.
Mobile-First Design
The entire platform is built mobile-first with a bottom navigation bar - designed to be opened on a phone courtside during live games.

Technical Architecture

The frontend is built with React 18 and Vite, deployed on Vercel. Supabase handles the PostgreSQL database, authentication-free anonymous sessions, and real-time subscriptions via Postgres Realtime channels. Every piece of live data - scores, picks, votes, rosters - syncs instantly across all connected clients without a page refresh.

The data layer uses a custom hooks pattern in useData.jsx. Each data type (teams, players, games, announcements) has its own hook that fetches on mount and subscribes to database changes for live updates. Each hook instance uses a unique channel name to avoid Supabase subscription conflicts when multiple components mount simultaneously. A fallback dataset ensures the UI renders immediately even before Supabase responds, so there is no loading flash on first visit.

Anonymous user sessions are tracked via localStorage IDs for features like voting and Pick'em - no login or account creation required, which was a deliberate choice to keep the barrier to participation as low as possible for casual fans. React Router v6 handles all client-side navigation. Stat visualizations use Recharts, and the entire visual design is built on CSS custom properties for consistent dark-mode-first theming across all pages.

What I Learned

This was my first serious project with React and my first time building a real-time data architecture. The biggest technical challenge was managing Supabase Realtime subscriptions at scale - early on, multiple components subscribing to the same channel caused conflicts and duplicate data. The solution was enforcing unique channel names per hook instance, which required understanding how Supabase manages WebSocket connections under the hood.

Building for a real audience - the SACC community actually using this during live games - pushed me to care about things I would have ignored in a class project: load time on mobile networks, graceful fallbacks when the database is slow, and UI decisions that make sense to someone who just wants to check the score quickly. That gap between "it works" and "it works for real people in real conditions" was the most valuable thing this project taught me.

The ECE 1100 Discovery Project framework challenged me to not just build something, but to investigate a technical topic at the frontier of what I know. For me, that was real-time web architecture: how do you design a system where data consistency, latency, and user experience all have to be solved simultaneously? This project is my answer to that question.
