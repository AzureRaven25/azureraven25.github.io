Overview

binge. is a social media-tracking web app where users build a personal profile around everything they watch and read. You can browse a catalog of movies, TV shows, and books, rate them, track your watchlist status, and discover what your friends are into — all in one place with a clean, dark aesthetic.

It sits somewhere between Letterboxd and Reddit: part personal journal, part social network, with a discussion forum layered on top. The social side is central — users follow each other, see trending content, collaborate on ranked lists with live voting, and can even watch things together in synchronized Watch Rooms. An AI chatbot (powered by Groq) is embedded throughout for recommendations, and an auto-moderator keeps the forum clean without manual intervention.

Features

Content Browsing
Movies, TV shows, and books with search, filtering, and detail modals. Live TV page with channel browsing and embed player. Global search (Cmd+K) across all media.
Personal Tracking
Watchlist with per-item status (Plan to Watch / Watching / Watched). Progress tracking per TV show (season + episode) and book (chapter/page). Star ratings and numeric scores.
Social Graph
User profiles with avatar, bio, follower/following counts, and public watchlist. Follow/unfollow, following feed, trending page, and notifications bell.
Collaborative Lists
Create named watchlists (public or private), invite collaborators by username, vibe-vote (upvote/downvote) to surface a consensus pick, and share via public URL.
Discussion Forum
Sub-forums with custom icons and rules. Posts with flair tags, hashtags, and markdown. Nested comment threads with voting, editing, and collapsing. Spoiler overlays and a report system.
Watch Together
Synchronized Watch Rooms where multiple users watch the same content simultaneously, with a live room chat alongside the embedded player.
AI Features
Groq-powered chatbot for recommendations and discovery. Automatic forum moderation that blocks offensive posts before they publish — no manual review needed.
Admin Panel
User management (view, search, promote, ban), media request approvals, and an analytics dashboard with usage stats, chat logs, and error tracking.

Tech Stack

Frontend

React 19 (Create React App) with React Router 7 for client-side routing. All UI is custom CSS — no component library — with a consistent dark design system. Communicates with Supabase directly via @supabase/supabase-js and falls back to the Express API for features requiring server-side logic.

Backend

Node.js / Express 5 server handling routes that need secrets or server-side processing: Groq AI chat and moderation, forum operations, notifications, admin actions, and watch rooms. Supabase (Postgres + Auth + Row Level Security) is the primary database.

Data Sources

Media catalogs seeded from TMDB, Plex, Internet Archive, and Open Library using custom scraper scripts. Books sourced from Open Library and Internet Archive APIs.

Deployment

Frontend deployed on Vercel. Express server runs as a separate service. Supabase manages authentication, all media data, ratings, watchlists, lists, forum content, and the social graph.

Design Decisions

The entire UI is hand-rolled CSS — no Tailwind, no component library. This was a deliberate choice to maintain full control over the dark aesthetic and ensure every interaction felt native to the platform's identity. The design system uses a consistent set of tokens (spacing, color, typography) applied uniformly across 40+ distinct views.

The AI moderation system runs as middleware on the Express server, intercepting post and comment creation requests and routing them through Groq before they touch the database. This keeps moderation latency low and completely invisible to the end user — offensive content simply never appears, rather than being flagged after the fact.

Supabase Row Level Security policies enforce data access at the database layer, meaning even if the application layer had a bug, private lists and user data would remain protected. The admin panel and developer lab are gated by a role check that Supabase RLS enforces server-side.
