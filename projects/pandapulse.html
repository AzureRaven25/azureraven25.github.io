Overview

PandaPulse is a full-featured job search and hiring platform built in Django for CS 2340 (Objects and Design) at Georgia Tech. The platform connects job seekers with employers through structured profiles, job listings, and an application pipeline — functioning similarly to LinkedIn but built entirely from scratch by our team. It serves three distinct user roles: Job Seekers, Recruiters, and Administrators.

The project placed 3rd out of all submissions across both class sections, earning our team a final exam exemption. Beyond the 15 required user stories, we designed and implemented over 20 original features that pushed the platform well beyond the baseline requirement.

Original Features

Panda Paul — AI Chatbot Mascot
A panda mascot chatbot that lets logged-in users navigate the site, update profile info, apply to jobs, and message people through natural conversation.
Pulses — Short-Form Video Feed
Any logged-in user can post short videos to promote themselves or their company. Users can like pulses; posters can delete their own content.
Offer Comparison Tool
Applicants compare job offers side-by-side with custom-weighted categories: salary, location, visa sponsorship, and benefits.
Interview Calendar System
Applicants schedule interviews from employer-set time slots, fully integrated into the workflow on both sides.
Response-Time SLA Badge
Each employer displays a "responds in ~N days" badge on job listings so applicants can set expectations before applying.
Resume Auto-Parser
Uploaded resumes are parsed into profile fields with skill normalization for better job matching accuracy.
Application Viewed Indicator
Applicants can see in real time when an employer has opened and viewed their application.
Applicant Comparison Tool
Recruiters compare multiple applicants for a role side-by-side and re-rank them against hiring priorities.
Saved Search Alerts
Recruiters save candidate searches and receive alerts when new matching profiles appear.
Company Culture Pages
Company profiles with culture descriptions, perks, and an office location map so candidates can evaluate fit before applying.
Dark Mode
Full site dark/light mode toggle that persists per user across sessions.
Skill Endorsements
Employers endorse applicant skills after interviews, adding social proof to candidate profiles.
One-Click Follow-Up
Applicants send a professional follow-up message after a set number of days using pre-built templates.
Rejection Feedback Templates
Structured rejection messages with optional personalized notes so candidates always receive meaningful feedback.
Archived Rejected Applicants
Employers archive rejected candidates to revisit them if new positions open up, preserving the talent pool.
Interview Feedback Workflow
Employers log structured internal feedback and decision rationale for each candidate after interviews.
Confetti on Application Submit
Application submissions are celebrated with confetti and fanfare to make the experience feel rewarding.
Account Recovery Flow
Forgot username and password flows that send recovery links to the associated email address.

Technical Approach

The platform was built entirely in Django, leveraging its ORM, class-based views, and template system throughout. The project emphasized the object-oriented design principles at the core of CS 2340 — clean model separation, reusable components, and a structured codebase built to scale. The map features use Google Maps to pin job locations and cluster applicants geographically.

The Pulses feed, Panda Paul chatbot, and offer comparison tool were all designed and built entirely by our team without external starter code. The dark mode system uses a per-user preference stored in the database and applied via CSS class switching on the root element.
